
========================================== Stars / Objects ====================================

A star object contains the basic information on a single star of binar system and can be identified by multiple identifiers/names. Identifiers are stored in a separate table. 
Tags can be added to stars, f.x. to split them in separate groups (UVES,MERCATOR,...)

/api/stars/stars/
   get:
      returns a list of all stars/objects in the database
      
   post:
      create a new star/object

      
/api/stars/stars/<id>
   get:
      returns this star
      
   delete:
      deletes this star
      
   put:
      updates this star

      
/api/stars/tags/
   get:
      returns a list of all tags in the database
      
   post: 
      create a new tag

   
/api/stars/tags/<id>
   get:
      returns this tag
      
   delete:
      deletes this tag
      
   put:
      updates this tag

   
/api/stars/identifiers/
   get:
      returns a list of all identifiers
      
   post: 
      create a new identifier


/api/stars/identifiers/<id>
   get:
      returns this identifier
      
   delete:
      deletes this identifier
      
   put:
      updates this identifier

========================================== Analysis ====================================

A dataset contains the result of a specific analysis method. A method can be anything, f.x. RV curve, SED fit, ... 
A parameter belongs to a star, and can belong to a specific dataset or be an average or derived parameter.

/api/analysis/methods/
   get:
      returns a list of all methods
      
   post: 
      create a new method
   
   
/api/analysis/methods/<id>
   the standard get/put/delete
   

/api/analysis/datasets/
   get:
      returns a list of all datasets
      
   post: 
      create a new dataset
      Doesn't work. Fileupload is not yet implemented!
      
      
/api/analysis/datasets/<id>
   standard get and delete. Can't upload files, so filechanges are not possible
   

/api/analysis/parameters/
   get:
      returns a list of all parameters
      
   post: 
      create a new parameter

      
/api/analysis/parameters/<id>
   standard get/put/delete
